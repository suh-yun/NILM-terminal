# 건물 전력 모니터링 

요구사항의 1단계인 **강의실 목록 → 강의실 선택 → 전력 소모 중인 제품 리스트 표시**까지 동작하는 정적(HTML/JS) 데모입니다.

## 파일별 핵심 요약

1. app.js : 웹 브라우저와 통신하거나 웹사이트의 전체적인 로직을 담당하는 메인 자바스크립트
2. style.css : index.html에서 화면을 꾸미는 디자인 파일
3. data.js : 전력 사용량이나 상태 값 같은 데이터를 따로 관리하고 처리
5. index.html : 눈으로 보이는 화면 (버튼, 그래프) 등을 생성
4. arduino_main : 아두이노 기기에서 돌아가는 가장 기본이 되는 설정 코드
6. main loop: 아두이노가 멈추지 않고 계속 반복하며 센서 값을 읽거나 명령을 대기하는 핵심 루프
7. 전력차단로직 : 특정 조건이 되면 전력을 끊어야 한다고 판단하는 알고리즘
8. 전력차단명령 : 로직에서 차단하라는 메세지를 하드웨어에 보내는 코드



## 웹사이트 기능

- 건물 메인 화면
  - 아산공학관/신공학관 현재 총 소비 전력 표시(데모 데이터 합산)
- 강의실 화면 (`#/classrooms/...`)
  - 건물 탭(아산공학관/신공학관)
  - 층 버튼(예: 1~4층, B2~5층) 및 검색
  - 강의실 리스트 표시 → 강의실 상세로 이동
- 강의실 상세 화면 (`#/classroom/{buildingId}/{classroomId}`)
  - 해당 강의실의 **전력 소모 중인 제품(isOn && currentW > 0)** 리스트 표시
  - 제품별 메모 입력 및 저장 (브라우저 `localStorage`에 저장)
  - 제품별 잠금/해제 (브라우저 `localStorage`에 저장)
  - 기기 상세 화면 (`#/device/{buildingId}/{classroomId}/{deviceId}`)
    - 전원 켜기/끄기 (데모: `localStorage`에 저장)
    - 작업중/작업아님 표시 및 토글 (데모: `localStorage`에 저장)
  - 차단 버튼 UI는 준비되어 있고, 현재는 비활성(알고리즘/서버 API 연결 예정)
- 커뮤니티 화면 (`#/community`)
  - 프로필 설정: **경비 / 학생(번호) / 교직원(번호) / 교수(이름)**
  - 글 작성: 공지사항/자유글
  - 글 목록: 전체/공지/자유 필터
  - 내가 쓴 글 삭제
  - 데이터 저장: 브라우저 `localStorage` (데모)

🔹 STEP 0. 시스템 목적 
목표: 콘센트에 꽂힌 가전의 전류 파형만 보고
켜짐 / 꺼짐 / 대기 상태를 판단하고
어떤 가전인지 식별한 뒤
차단 로직에 따라 자동으로 전력을 차단

ex) 현재 연결된 기기는 전기포트 → 30분이 지났으니 사용 중이 아닐 거라고 추측  → 전력 차단


## 전체적인 차트
🔹 STEP 1. 전류 측정 (하드웨어 입력)
1️⃣ 전류 측정 멀티탭
가전제품은 전류 측정 멀티탭에 연결
멀티탭 내부의 전류 센서가 전류 파형을 인식

2️⃣ ESP32
ADC로 전류 샘플링
예:샘플링 주파수: 5~10kHz
일정시간(0.5~1초)씩 데이터 수집
시계열 데이터로 전류 수집
i(t) = [0.12, 0.15, 0.18, ...]  

🔹 STEP 2. 데이터 전송 (ESP32 → 서버/PC)
ESP32가 측정한 전류 데이터를
Serial 또는 Wi-Fi (HTTP / MQTT)로 컴퓨터(서버)로 전송\
 → 측정 및 전달만 실행

🔹 STEP 3. 전처리 (신호 정리)
서버/PC에서 수행

3-1. 오프셋 제거
i0(t) = i(t) − mean(i)

3-2. 프레임 분할
0.5~1초 단위로 신호 분할

3-3. 전류를 P = V*I 공식을 이용해 전력으로 변환 후 전류 및 전력 데이터를 웹사이트의 각 부분으로 전송


🔹 STEP 4. 특징 추출 
머신러닝을 위해 각 프레임마다 시간 특징으로 변환하여 csv 파일로 변환

시간 영역 특징
RMS (소비 전력 수준)
Crest Factor (피크 성분)
주파수 영역 특징 (FFT)

H1 (60Hz)
H3, H5, H7
H3/H1, H5/H1 같은 비율

결과 예시 (CSV 한 줄)
RMS,H1,H3,H5,H7,Label
0.83,1.24,0.31,0.12,0.05,HairDryer


🔹 STEP 5. 머신러닝 학습 
입력 (X)
[RMS, H1, H3, H5, H7]

출력 (y)
가전 이름 (pot, charger, airclean, ...)

사용 모델
RandomForestClassifier (머신러닝)

→ 정확도 ~93% 달성함
머신러닝 파일: nilm_rf_model.pkl
              nilm_scaler.pkl


👉 여기까지 = 학습 단계 끝

🔹 STEP 6. 실시간 추론 (실사용 단계)
6-1. 새로운 전류 입력
ESP32 → 서버로 데이터 이동
새 전류 프레임

6-2. 전처리 + 특징 추출
(학습 때랑 동일하게 진행)

6-3. 모델 예측
label = model.predict(X_new)
proba = model.predict_proba(X_new)

출력 예:
Indu (확률 0.92)


🔹 STEP 7. 상태 판단 (NILM 로직)
머신러닝 결과를 규칙으로 해석
예:
RMS < threshold → Standby
RMS ≈ 0 → Off
RMS > threshold → Active

또는
Microwave + Standby 10분 이상 → 차단


🔹 STEP 8. 전력 차단 (하드웨어 제어)
차단 멀티탭
ESP32 + 릴레이 모듈
릴레이 ON/OFF로 AC 전원 차단


[전체 구조]

[벽 콘센트]
   ↓
[차단 멀티탭 (릴레이)]
   ↓
[전류 측정 멀티탭]
   ↓

ESP32가 전류를 측정 → 서버에서 머신러닝으로 가전과 상태를 판단 →
